# 📊 PostgreSQL & Greenplum & Airflow: Лабораторные работы и отчеты

Этот репозиторий содержит PDF-отчеты и файлы исходного кода по всем практическим работам в рамках изучения MPP-СУБД **Greenplum** (на базе PostgreSQL) и оркестратора **Apache Airflow**. Здесь собраны материалы от базового DDL до развертывания кластера, оптимизации производительности и построения автоматизированных пайплайнов обработки данных.

### Модуль 3: Оркестрация и автоматизация с Apache Airflow

#### 🔹 Практическая работа №3.1
**Первое знакомство с Airflow. Создание DAG'а**
- Установка и настройка Apache Airflow
- Сборка и отладка первого DAG'а
- Исправление ошибок в коде, форматирование и расширение логики
- Добавление дополнительных шагов (тасок) и настройка цепочки зависимостей между ними
- *Файл:* `my_test_dag.py`

#### 🔹 Практическая работа №3.2
**Архитектура и концепции Airflow. Создание пайплайна**
- Разработка трех DAG'ов, взаимодействующих друг с другом:
    1.  **DAG 1:** Запуск по расписанию (каждые 40 минут). Состоит из двух `DummyOperator`, выполняющихся последовательно.
    2.  **DAG 2:** Запуск по расписанию (каждые 20 минут). Состоит из трех `BashOperator` (с командой `sleep`), выполняющихся параллельно.
    3.  **DAG 3:** Запуск по расписанию (каждые 55 минут). Использует `ExternalTaskSensor` для ожидания успешного завершения:
        -   Конкретной таски из DAG 1.
        -   Всего DAG 2 целиком.
- *Файлы:* `pyatunin_dags.zip`

#### 🔹 Практическая работа №3.3
**Обработка информации с удаленных источников**
- Получение и обработка инкрементальных данных (на примере Citi Bike trip data).
- Реализация DAG'а с backfill-стратегией (начиная с даты появления данных).
- Последовательное выполнение задач:
    - Скачивание помесячных ZIP-архивов с удаленного источника.
    - Разархивирование данных.
    - Фильтрация (поездки за первый понедельник месяца, очистка от «мусорных» данных).
    - Загрузка в партиционированную таблицу целевой БД (Greenplum, PostgreSQL и др.).
- Сохранение количества загруженных записей в Airflow Variables.
- *Файлы:* `dag_incremental_load.py`

#### 🔹 Практическая работа №3.4
**Продвинутый пайплайн**
- Интеграция Airflow с Greenplum и PostgreSQL.
- Создание сложного пайплайна с перекрестным взаимодействием DAG'ов.
- Реализация триггерного механизма: на целевой CSV-файл настраивается сенсор.
- Загрузка одного и того же файла в две разные СУБД (Greenplum и PostgreSQL).
- Передача параметров между дагами через XCom.
- Хранение магических переменных и строк подключения в Variables и Connections Airflow.
- *Файлы:* `dags.zip`
